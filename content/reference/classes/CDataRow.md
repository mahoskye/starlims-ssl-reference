# CDataRow

Represents one row in a [`CDataTable`](CDataTable.md).

`CDataRow` gives you row-level access to table data through [`CDataField`](CDataField.md) objects. Use it to read or assign field values, retrieve primary-key values, and build parameter arrays that match the SQL generated by [`CDataTable`](CDataTable.md).

## When to use

- When you need to read or change values in a single row from a [`CDataTable`](CDataTable.md).
- When you need a row's primary-key values for matching or persistence.
- When you want parameter arrays for `INSERT` or `UPDATE` statements built from the row.

## Constructors

No standalone SSL constructor is documented for this class.

Obtain `CDataRow` instances by accessing `Rows`, calling `Select(...)`, or calling `NewRow()` on a [`CDataTable`](CDataTable.md).

## Properties

| Name | Type | Access | Description |
|------|------|--------|-------------|
| `Table` | [object](../types/object.md) | read-only | Parent [`CDataTable`](CDataTable.md) for this row. |

## Methods

### `GetField`

Returns a field object for the requested column.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `vIndex` | any | yes | Column name, or a 1-based numeric column index. |

**Returns:** [`CDataField`](CDataField.md) ‚Äî The field object for the requested column.

**Raises:**

- **When `vIndex` is [`NIL`](../literals/nil.md):** `Argument vIndex cannot be null.`
- **When `vIndex` is not a string or integer:** `Argument vIndex is not string nor integer.`
- **When `vIndex` is an out-of-range integer:** `Index [<n>] is out of range (fields count=<count>)`
- **When `vIndex` names a column that does not exist:** `Column: [<name>] does not exist.`

### `GetIndex`

Returns the column position for a field name.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `sFieldName` | [string](../types/string.md) | yes | Column name to locate. |

**Returns:** [number](../types/number.md) ‚Äî Zero-based column position, or `-1` if the column is not found.

### `GetPKsValues`

Returns this row's primary-key values.

**Returns:** [array](../types/array.md) ‚Äî Primary-key values in primary-key order.

### `ToArray`

Returns all row values.

**Returns:** [array](../types/array.md) ‚Äî All column values in table order.

### `ToArrayForInsert`

Returns values for insert operations.

**Returns:** [array](../types/array.md) ‚Äî Non-binary column values in table order.

### `ToArrayForUpdate`

Returns values for update operations.

**Returns:** [array](../types/array.md) ‚Äî Non-binary, non-primary-key values followed by primary-key values.

### `GetProperty`

Resolves the parent table or a field object by name.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `sProp` | [string](../types/string.md) | yes | `"Table"` to get the parent table, or a column name to get that field object. |

**Returns:** [object](../types/object.md) ‚Äî The parent [`CDataTable`](CDataTable.md) for `"Table"`; otherwise the matching [`CDataField`](CDataField.md).

**Raises:**

- **When `sProp` names a column that does not exist:** `Column: [<name>] does not exist.`

**Base class:** [`object`](../types/object.md)

## Best practices

!!! success "Do"
    - Use `GetField()` when you need a specific column by name or position.
    - Use `GetField(...):Value` to read or assign the actual field value.
    - Use `GetPKsValues()` when you need the row's primary-key values.
    - Pair `ToArrayForInsert()` with `CDataTable:GetInsertSql()`.
    - Pair `ToArrayForUpdate()` with `CDataTable:GetUpdateSql()`.
    - Convert `GetIndex()` results before passing them to `GetField()`.

!!! failure "Don't"
    - Treat `GetIndex()` as a 1-based SSL array index ‚Äî it returns a zero-based position and `-1` when the column is missing.
    - Treat row property access as the raw field value ‚Äî it resolves to a [`CDataField`](CDataField.md); use `:Value` for the actual data.
    - Use `ToArray()` for insert or update parameter binding ‚Äî it includes all columns and does not match the filtered ordering used by the table SQL helpers.
    - Assume `ToArrayForUpdate()` excludes primary keys ‚Äî it appends primary-key values after the non-primary-key values to match the generated `WHERE` clause.
    - Expect binary columns in `ToArrayForInsert()` or `ToArrayForUpdate()` ‚Äî both helpers omit them.

## Caveats

- Accessing a property by column name (e.g. `oRow:status`) returns a [`CDataField`](CDataField.md) object, not the raw field value. Use `:Value` to read or assign the actual data.

## Examples

### Read a field by name or index

Uses `GetIndex()` to locate the column (zero-based), adds 1 to convert to a 1-based index, then calls `GetField()` to get the field. Returns early when the column is missing (`GetIndex` returns `-1`).

```ssl
:PROCEDURE ReadStatusField;
	:PARAMETERS oRow;
	:DECLARE nColIndex, oField;

	nColIndex := oRow:GetIndex("status");

	:IF nColIndex == -1;
		:RETURN "";
	:ENDIF;

	oField := oRow:GetField(nColIndex + 1);

	:RETURN oField:Value;
:ENDPROC;

/* Usage;
DoProc("ReadStatusField", {oRow});
```

### Create and populate a new row

Calls `NewRow()` on the table to create an empty row, assigns field values through `GetField():Value`, then adds the row to the table with `AddRow()`.

```ssl
:PROCEDURE AddSampleRow;
	:PARAMETERS oTable, sSampleID, sStatus;
	:DECLARE oRow;

	oRow := oTable:NewRow();
	oRow:GetField("sample_id"):Value := sSampleID;
	oRow:GetField("status"):Value := sStatus;

	oTable:AddRow(oRow);

	:RETURN oRow;
:ENDPROC;

/* Usage;
DoProc("AddSampleRow", {oTable, "S-001", "Logged"});
```

### Save a row with the table SQL helpers

Pairs `ToArrayForInsert()` or `ToArrayForUpdate()` with the matching SQL string from the table. Both helpers produce arrays whose column ordering matches what the table SQL methods generate, making them safe to pass directly to [`RunSQL`](../functions/RunSQL.md).

```ssl
:PROCEDURE SaveRow;
	:PARAMETERS oTable, oRow, bInsert;
	:DECLARE sSQL, aParams, bSuccess;

	:IF bInsert;
		sSQL := oTable:GetInsertSql();
		aParams := oRow:ToArrayForInsert();
	:ELSE;
		sSQL := oTable:GetUpdateSql();
		aParams := oRow:ToArrayForUpdate();
	:ENDIF;

	bSuccess := RunSQL(sSQL,, aParams);

	:RETURN bSuccess;
:ENDPROC;

/* Usage;
DoProc("SaveRow", {oTable, oRow, .T.});
```
